# fork부터 PR까지의 흐름

1. 원본 레포지토리를 fork하면 해당 내용이 나의 개인 레포지토리에 복사된다.

2. 복사된 레포지토리의 주소를 single--branch 옵션을 주어 clone하여 브랜치를 분기한다.

3. 내가 만든 브랜치에서 원본은 훼손시키지 않고 따로 코드를 수정한다.

4. 수정한 내용에 대해 git add를 하고 git commit 한 후, 내가 만든 브랜치에 git push한다.
    - 여기까지 원본의 내용은 일체 변함이 없으며, fork를 통해 복사된 나의 개인 레포지토리에서만 수정된다.

5. 원본 레포지토리에 Pull Request를 보내 나의 코드를 원본 레포지토리의 브랜치로 보낼 수 있도록 한다.

6. Pull Request가 받아들여져도 merge를 하지 않으면 원본 내용은 수정되지 않는다.
